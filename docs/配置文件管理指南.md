# Python 配置文件管理指南

## 配置文件对比：Java vs Python

### Java 常见配置文件
```
src/main/resources/
├── application.yml          # 主配置文件
├── application-dev.yml      # 开发环境配置
├── application-prod.yml     # 生产环境配置
└── application.properties   # 或使用 properties 格式
```

### Python 对应方案
```
项目根目录/
├── config.yaml             # 应用配置（对应 application.yml）
├── config.dev.yaml         # 开发环境配置（可选）
├── config.prod.yaml        # 生产环境配置（可选）
├── .env                    # 敏感环境变量（不提交）
├── .env.example            # 配置模板（提交）
└── settings.py             # 配置加载逻辑（可选）
```

## 文件说明

### 1. .env（敏感配置 - 不要提交）
```bash
# API 密钥
KIMI_API_KEY=sk-xxx...
DATABASE_PASSWORD=secret123
SECRET_KEY=random-secret-key

# 第三方服务
AWS_ACCESS_KEY=xxx
AWS_SECRET_KEY=yyy
```

**特点**：
- 包含敏感信息（密钥、密码）
- 每个环境有不同的值
- 通过 `.gitignore` 忽略
- 使用 `python-dotenv` 库加载

### 2. .env.example（配置模板 - 要提交）
```bash
# API 密钥（请填写你的实际密钥）
KIMI_API_KEY=your_key_here
DATABASE_PASSWORD=your_password_here

# 第三方服务
AWS_ACCESS_KEY=your_access_key
AWS_SECRET_KEY=your_secret_key
```

**特点**：
- 不包含真实的敏感值
- 提交到仓库供团队参考
- 新成员复制为 `.env` 并填入实际值

### 3. config.yaml（应用配置 - 可以提交）
```yaml
# 应用配置（非敏感）
app:
  name: "Hello Agent"
  version: "0.1.0"
  debug: false

server:
  host: "0.0.0.0"
  port: 8080
  timeout: 30

database:
  host: "localhost"
  port: 5432
  name: "myapp"
  # 密码从环境变量读取，不写在这里

logging:
  level: "INFO"
  format: "json"
```

**特点**：
- 不包含敏感信息
- 可以安全提交到仓库
- 类似 Java 的 `application.yml`

## Python 加载配置的方式

### 方法 1: 使用 python-dotenv（推荐）
```python
from dotenv import load_dotenv
import os

# 加载 .env 文件
load_dotenv()

# 读取环境变量
api_key = os.getenv("KIMI_API_KEY")
```

### 方法 2: 使用 PyYAML 读取配置文件
```python
import yaml

with open("config.yaml", "r") as f:
    config = yaml.safe_load(f)

# 使用配置
port = config["server"]["port"]
```

### 方法 3: 使用 Pydantic Settings（现代方案）
```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    kimi_api_key: str
    database_password: str
    
    class Config:
        env_file = ".env"

settings = Settings()
```

## 最佳实践总结

| 配置类型 | 存放位置 | 是否提交 | 示例 |
|---------|---------|---------|------|
| API 密钥 | `.env` | ❌ | `KIMI_API_KEY=sk-xxx` |
| 数据库密码 | `.env` | ❌ | `DB_PASSWORD=secret` |
| 配置模板 | `.env.example` | ✅ | `KIMI_API_KEY=your_key_here` |
| 应用配置 | `config.yaml` | ✅ | `port: 8080` |
| 服务器地址 | `config.yaml` | ✅ | `host: localhost` |
| 日志级别 | `config.yaml` | ✅ | `level: INFO` |

## 常见问题

### Q1: 为什么不把所有配置都放在一个文件？
A: **安全性**。敏感信息（密钥）和非敏感信息（端口号）应该分开。密钥需要保密，而端口号可以公开。

### Q2: 团队成员如何快速配置环境？
A: 
1. 克隆项目
2. 复制 `.env.example` 为 `.env`
3. 在 `.env` 中填入实际的 API Key
4. 运行 `poetry install` 安装依赖

### Q3: 不同环境（开发/测试/生产）如何管理配置？
A: 
- **方案 1**: 使用不同的 `.env` 文件（`.env.dev`, `.env.prod`）
- **方案 2**: 使用环境变量指定配置文件：`ENV=prod python app.py`
- **方案 3**: 在部署平台（如 Docker、K8s）中注入环境变量

### Q4: 如果不小心提交了 .env 怎么办？
A:
```bash
# 1. 从 Git 历史中移除文件
git rm --cached .env
git commit -m "Remove .env from git"

# 2. 立即更换所有泄露的密钥！
# 3. 确保 .gitignore 中包含 .env
```

## Python 配置库推荐

1. **python-dotenv** - 加载 `.env` 文件（最流行）
2. **pydantic-settings** - 类型安全的配置管理（现代推荐）
3. **dynaconf** - 支持多环境、多格式的配置管理
4. **configparser** - Python 内置，类似 Java Properties
5. **PyYAML** - 解析 YAML 配置文件

## 总结

Python 的配置管理思路与 Java 类似，但更灵活：

- **Java**: 主要使用 `application.yml`/`properties`，敏感配置通过环境变量或配置中心
- **Python**: 使用 `.env` + `config.yaml` 组合，职责更明确

核心原则：
- ✅ **敏感信息**（密钥、密码）→ `.env`（不提交）
- ✅ **配置模板** → `.env.example`（提交）
- ✅ **应用配置**（端口、超时）→ `config.yaml`（提交）
- ✅ **依赖管理** → `pyproject.toml` + `poetry.lock`（提交）
